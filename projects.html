Dominion Rust

One of my favorite board games is dominion. In an effort to overdo a final class project, I decided to try and write the game in rust for educational purposes, using the terminal.

I implemented about half of the cards in "First game," along with a CPU opponent that plays a simple strategy called "Big money"

Considering the standards for this project was a terminal-style command interface, and I was working alone, I'm proud of the outcome.

A child wandering the stars

This is an abandoned Game design project that tested the limits of how far I could push physics calculations and proper gravity handling.

As the only programmer on this project, I wrote the entire player movement, physics, gravity, and rotation from scratch, so a player could move around a 2d sphere.

This project made me familiar with cross-class signaling, modern application architecture, and some nuances of OpenGL and resolution handling.

Godot isn't built for games that center around spherical coordinates without heavy modification, so the project is on hold until I can aquire some skills, and make a return.

This game also had a number of lovely art assets and animations done by someone who doesn't have a portfolio link, so I can't show off their art! :(
